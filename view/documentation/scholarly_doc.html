1.Introduction

1.1 Purpose

The purpose of this document is to give you information and guidance on the implementation, functionalities and usage of a service providing app that hosts different services. The app stores, manipulates and displays information on the services hosted, giving the users possibility to edit, view, download and responsively interact with the interface in order to be able to post or book services directly from the app.

1.2 Product Scope

The app is intended to grow the business and improve some areas based on insider data and given statistics, as a result of information collected from its users. The app will serve as an intermidiate for hosting different kind of services, store and update data on business info, service providers info, bookings; we give the consumers accurate information on the current disponible services, trainers, free time-lots and a lot of other information, so that the customer could visualize and decide what's best for him in advance.
The main admin will have access to the insider data and statics, be capable of adding new employee accounts.
The human service providers will have a special account and will be able to vizualize data on the company they work for. The basic user will have the possibility of vizualize the general information on the website and book services.
It is good to know that the app doesn't dispose any payment method. After the booking of the service, the client will be given a personal code with which he/she could go to the service location and pay directly from there.

2.Overall Description

2.1 Product Functions

Major features of our program include:

  1. Booking a service
  2. Introducing a new employee
  3. Downloading service data in CSV, XML, HMTL and PDF
  4. Querying the database to get statistics on all the businesses
  5. Review a service after booking it
  6. Authentication
All these features will be described in the paragraphs below.

2.2 User Classes and Characteristics

The program was designed to be used by an average user. The functionalities are easy and straightforward, so it is not needed any preliminary guidance.
- Frequency of use. An average user should access the site twice per week to check for updates and new services to buy. It should be your to-go app on a regular basis for standard services.
- Subset of product functions used. The product has basic functions like authentication, browsing, booking and reviewing. For the general user they are easy to use. The statistics part is accessible only by admin, even though he should have some basic knowledge of how the system works and operates.
- Education level and expirience. The general user should be at least in middle school in order to use the app, whereas the admin should have working knowledge in order to operate in the statistics page.

2.3 Operating Environment

This is a web-app which will be disponible from any currently known web-browser(Chrome, Mozila Firefox, Google Chrome, Safari) and on any operating system which supports them(Windows, IOS, Android, Linux).

2.4 Assumptions and Dependencies

For our app we will use Google Charts in order to make the visual representation of the diagrams more attractive. Because we will make complex diagrams of different types (pie, bar, column) and they will manipulate complex data, it was decided to use this library instead of doing it from skratch.

4. System features

4.1 Add employee

This feature, that can be accesed only by admin, involves creating a new user account for an employee. The admin fills in the required fields (username, email, password), click on the "Add" button and, if the information is valid a new user account of a specific department(massage, fitness, uber) is created.

4.2 Log in

The employee-user authentication can be performed by accessing "Log in" on Home page. If the introduced data is correct, the user will be redirected to program page of department he is part of, for example a massage employee will see all reservation for masage.

4.3 Booking

Each client can book any of the services presented, within the time limits or available subscriptions. Depending on the type of service, additional data may be requested. For example to order an Uber it is necessary to specify the point of departure and the destination.

4.4 Statistics

The statistics page can be used only by admin. Statistics are generated based on stored data(in database) and choosed criteria (type of service, time zone, etc). Charts obtained from this process can be downloaded in HTML,CSV, PDF and XML format.

4.5 Admin

  • Maintain proper record of available services and customers in database;
  • Add new employees;
  • Manage all software;
  • Access al feedbacks received
  • Access statistics and download them
